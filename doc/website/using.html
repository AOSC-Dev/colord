How do I use colord?

Using the command line

The colormgr text-mode program allows you to interact with colord on the command line. For example:

Using graphical tools:

gnome-color-manager provides a rich set of GTK tools for automatically managing color devices and profiles. See the screenshots page for more details.

Using libcolord:

The libcolord gobject library wraps the DBUS interface in a nice glib-style API. This makes designing programs that use libcolord can concentrate on core functionality rather that the DBUS and colord internals. CdClient in libcolord can be used as easily as: 

CdClient *client;
GError *error;
GPtrArray *devices;
client = cd_client_new ();
devices = cd_client_get_devices_sync (client, cancellable, &error);
g_object_unref (client);
g_ptr_array_unref (devices);

Using the raw DBUS API:

Using the DBUS methods and signals directly means that no glib or gobject dependency is needed. This is not difficult, although does require more code than just using libcolord. See this file for an example raw-dbus.c. The latest interface is available in the source tree or on-line.
